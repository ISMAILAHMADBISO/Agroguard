# Generated API client must send credentials for cookie auth

The orval-generated React Query client (`@workspace/api-client-react`) routes every
request through the hand-written mutator `custom-fetch.ts` (`customFetch`). That
mutator's `fetch()` call must pass `credentials: init.credentials ?? "include"`,
otherwise the browser never sends the session cookie on hook-based requests.

**Why:** symptom was login working (the auth context uses *raw* fetch with
`credentials:"include"`) while every React Query hook 401'd — empty
farmer/device/staff lists and "authentication required" on create. Root cause was
the mutator omitting credentials, not the server session/CORS/CSRF config (that was
already correct: trust proxy, Secure/SameSite=None on Replit, CORS allowlist,
Origin CSRF guard).

**How to apply:**
- `custom-fetch.ts` is the mutator referenced by `orval.config.ts` — it is NOT
  regenerated by codegen, so editing it is safe and survives `pnpm run codegen`.
- After editing it, run `pnpm run typecheck:libs` (it's a composite lib) before the
  frontend picks up the change.
- Cookie default of `include` is safe here because the server enforces a credentialed
  CORS allowlist + Origin/Referer CSRF guard on non-GET. Native/Expo clients use
  bearer tokens via `setAuthTokenGetter` and are unaffected (can override per-call).
